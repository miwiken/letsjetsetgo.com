# /verify

Non-blocking verification report — run specialized agents in parallel against a PR, post one consolidated comment.

**Replaces the Copilot review + /parry loop.**

## Usage

```
/verify 42                    # Run against PR #42
/verify                       # Run against current branch's PR
/verify --url https://...     # Specify preview URL explicitly
/verify --repo org/repo       # Target a specific repo
/verify --agents sast,copilot # Run only specific agents
```

## Process

### 1. Resolve the PR

```bash
gh pr view {number} --json number,title,url,headRefName,baseRefName
# Or from current branch
gh pr view --json number,title,url,headRefName,baseRefName
```

### 2. Gather context

```bash
gh pr diff {number} --name-only
gh pr diff {number}
```

### 3. Load config

Read `.verify.yml` from this project's root. If missing, use defaults:
- **Always on:** copilot, sast, docs-freshness
- **On if preview URL detected:** accessibility, responsive, discovery
- **Opt-in via `.verify.yml`:** dast, visual-regression, second-opinion (sends diff to third-party API)

### 4. Run verification

Load the verify skill from work-work (`skills/verify/SKILL.md`) and follow its workflow. The skill dispatches agents in parallel, collects structured reports, and consolidates into one PR comment organized by severity.

Agent definitions live in `skills/verify/agents/` in the work-work repo.

### 5. Report

```
Verification — PR #{number}: {title}
{N} findings | {critical} critical, {warning} warning, {info} info
Agents: {list} | Skipped: {list} | Duration: {N}s
```

## Rules

- **Non-blocking.** Findings are advisory, not a merge gate.
- **One PR comment.** Organized by severity, not by agent.
- **No AI attribution.** No "generated by" or similar in the comment.

- **Missing prerequisites = `skipped`, runtime failure = `error`.**
